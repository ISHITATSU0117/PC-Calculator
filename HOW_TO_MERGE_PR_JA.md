# プルリクエストのマージ方法（詳細ガイド）

## ❓ "Merge pull request" ボタンはどこにありますか？

このガイドでは、GitHubでプルリクエスト（PR）をマージする具体的な手順を説明します。

---

## 📍 ステップ1: プルリクエストのページを開く

### 方法A: GitHubの通知から
1. GitHubにログイン
2. 画面右上のベルアイコン🔔をクリック
3. 通知リストからこのPRをクリック

### 方法B: リポジトリから直接
1. GitHubでこのリポジトリを開く
   - URL: https://github.com/ISHITATSU0117/PC-Calculator
2. 上部メニューの **「Pull requests」** タブをクリック
3. PRリストから該当のPRをクリック
   - 例: "#7 Fix issue with changes not reflecting on webpage"

---

## 📍 ステップ2: PRページで下にスクロール

PRページを開いたら：

1. **ページを下にスクロール**してください
2. コメント欄や変更内容（Files changed）を通り過ぎます
3. **一番下**までスクロールします

---

## 📍 ステップ3: マージセクションを見つける

ページの一番下に、以下のような**緑色のセクション**が表示されます：

```
┌─────────────────────────────────────────────┐
│ ✓ This branch has no conflicts with the     │
│   base branch                                │
│                                              │
│ ┌─────────────────────────────┐             │
│ │  Merge pull request  ▼      │ ← ここ！   │
│ └─────────────────────────────┘             │
│                                              │
│ Merge using the merge commit strategy       │
└─────────────────────────────────────────────┘
```

### 🎯 ポイント:
- **緑色の大きなボタン**です
- ボタンには "**Merge pull request**" と書かれています
- ボタンの右に小さな▼（ドロップダウン）があります

---

## 📍 ステップ4: マージを実行

### 4-1. Merge pull requestボタンをクリック

ボタンをクリックすると、入力欄が表示されます：

```
┌─────────────────────────────────────────────┐
│ Merge pull request #7 from ...              │
│                                              │
│ ┌─────────────────────────────────────────┐ │
│ │ Add comprehensive deployment...         │ │
│ └─────────────────────────────────────────┘ │
│                                              │
│ ┌─────────────────────────────────────────┐ │
│ │ [コミットメッセージの詳細（任意）]     │ │
│ └─────────────────────────────────────────┘ │
│                                              │
│ ┌─────────────────────────────┐             │
│ │  Confirm merge              │             │
│ └─────────────────────────────┘             │
└─────────────────────────────────────────────┘
```

### 4-2. Confirm mergeボタンをクリック

- コミットメッセージはデフォルトのままでOK
- **"Confirm merge"** ボタンをクリック

### 4-3. 完了！

マージが完了すると、以下のメッセージが表示されます：

```
✓ Pull request successfully merged and closed
```

---

## 🚨 ボタンが見つからない場合

### ケース1: ボタンが灰色（クリックできない）

表示されるメッセージ:
```
This pull request is closed
```

**対処**: すでにマージ済みまたはクローズ済みです。確認してください。

---

### ケース2: "conflicts"（競合）のメッセージが表示される

表示されるメッセージ:
```
✗ This branch has conflicts that must be resolved
```

**対処**: 
1. コンフリクトを解決する必要があります
2. 「Resolve conflicts」ボタンをクリック
3. または、開発者に連絡してください

---

### ケース3: マージボタンが表示されない

**考えられる理由**:
- 権限がない（リポジトリのコラボレーターではない）
- ブラウザのJavaScriptが無効
- ページが正しく読み込まれていない

**対処**:
1. ページをリロード（F5キー）
2. 別のブラウザで試す
3. リポジトリのオーナーに権限を確認

---

### ケース4: "Review required"（レビュー必要）のメッセージ

表示されるメッセージ:
```
At least 1 approving review is required
```

**対処**:
- リポジトリの設定でレビューが必須になっています
- リポジトリオーナーに連絡してください

## 📱 マージ方法の選択（上級者向け）

ボタン右の▼をクリックすると、3つの方法が選べます：

1. **Create a merge commit** （推奨）
   - すべてのコミット履歴を保持
   - デフォルトの方法

2. **Squash and merge**
   - すべてのコミットを1つにまとめる
   - 履歴をシンプルに保つ

3. **Rebase and merge**
   - リベースしてからマージ
   - 直線的な履歴を作成

**初めての方は**: デフォルトのまま（Create a merge commit）でOKです。

## ✅ マージ後の確認

マージが成功したら：

1. **GitHub Actions を確認**
   - リポジトリのActionsタブを開く
   - "Deploy to GitHub Pages"ワークフローを確認
   - 緑のチェックマーク✅が表示されるまで待つ（1〜3分）

2. **サイトにアクセス**
   - https://ishitatsu0117.github.io/PC-Calculator/
   - ブラウザのキャッシュをクリア: `Ctrl + Shift + R` (Win) / `Cmd + Shift + R` (Mac)

3. **変更が反映されていることを確認**

## ❓ よくある質問

**Q: 自分のアカウントでマージボタンが表示されません**

A: リポジトリのコラボレーター権限が必要です。リポジトリオーナー（ISHITATSU0117）に権限を依頼してください。

**Q: ボタンをクリックしても何も起こりません**

A: 
1. ブラウザのJavaScriptが有効か確認
2. ブラウザのコンソール（F12）でエラーを確認
3. ページをリロード（F5）

**Q: マージした後、元に戻せますか？**

A: はい。PRページの "Revert" ボタンで元に戻せます。ただし、新しいPRが作成されます。

**Q: どのブラウザが推奨ですか？**

A: Chrome、Firefox、Edge、Safariの最新版を推奨します。Internet Explorerは非対応です。
